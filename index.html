1. What the Internet Is

The Internet is a global system of interconnected computer networks that communicate using a common set of protocols called TCP/IP. At its heart it is a collection of hardware (routers, switches, servers, cables) and software (protocols, addressing systems) that allow information to move from one device to another across any distance.

Key points

  • Devices: computers, phones, servers act as endpoints.
  • Addresses: IP addresses uniquely identify devices on the network.
  • Packets: Data is broken into small chunks called packets for efficient transfer.

Example: When you request a webpage your computer collects the images, text, and scripts as separate packets, then reassembles them into the final page shown by your browser.

2. How Data Transfers & Devices Communicate

2.1 IP Addresses (IPv4 & IPv6)

IP addresses provide a logical address for devices. There are two versions in wide use:

IPv4
32-bit addresses, typically displayed as four decimal numbers separated by dots (e.g., 192.168.0.1). Limited address space (~4.3 billion addresses).
IPv6
128-bit addresses, displayed in hexadecimal separated by colons (e.g., 2001:0db8:85a3::8a2e:0370:7334). Vast address space, designed to replace IPv4 as adoption grows.

2.2 MAC Addresses

MAC (Media Access Control) addresses are hardware identifiers burned into network interfaces (NICs). They are used for local network delivery (LAN) and look like 00:1A:2B:3C:4D:5E. A packet on an Ethernet network uses MAC addresses to identify source and destination at layer 2.

2.3 ISPs and DHCP

ISPs (Internet Service Providers) connect you to the global Internet. Most home networks obtain an IP address from an ISP via DHCP (Dynamic Host Configuration Protocol), which assigns temporary addresses automatically.

2.4 DNS (Domain Name System)

DNS acts as the internet's phonebook — it maps human-friendly domain names (like example.com) to IP addresses. DNS records include A (IPv4), AAAA (IPv6), MX (mail), CNAME (aliases), and more.

DNS lookup process (click to expand)
  1. Your device queries the configured DNS resolver (often ISP or public DNS).
  2. If the resolver has the answer cached it returns it immediately.
  3. If not cached the resolver queries authoritative servers starting at the root, then TLD, and finally the domain's authoritative server to get the record.

3. Postal Delivery Analogy (Practical Comparisons)

To demystify networking, map it to a postal system:

Postal System Internet Equivalent Why it helps
House address IP address Helps you locate the recipient
Apartment number Port number Helps deliver to correct application (web, mail, etc.)
Post office ISP / Router Connects local mail to wider network
Sorting centers Routers Choose the best path for delivery

Using analogies simplifies abstract layers and shows how different parts of the network cooperate to route messages.

4. Step-by-Step: How a Website Is Accessed

  1. Type a domain name (e.g., www.example.com) in your browser.
  2. DNS lookup: Browser asks DNS resolver for the IP address of that domain. The resolver returns an A/AAAA record.
  3. TCP connection: Browser opens a TCP connection to the server IP (usually on port 80 for HTTP or 443 for HTTPS).
  4. TLS handshake (if HTTPS): Client and server agree on encryption keys using TLS so traffic is confidential and tamper-proof.
  5. HTTP request: Browser sends an HTTP GET request for the desired resource.
  6. Server response: Server returns status, headers, and the body (HTML, JSON, etc.).
  7. Render: Browser parses HTML, requests CSS and JS; builds DOM and render tree, executes scripts, and paints the page.

5. Role of Port Numbers

Ports let a single IP address host multiple services. They are 16-bit numbers (0-65535). Some well-known ports:

PortServiceExample Use
20/21FTPFile transfer
22SSHSecure shell login
25SMTPSending emails
53DNSDomain name lookups
80HTTPUnencrypted web traffic
443HTTPSEncrypted web traffic

Syntax: 192.168.1.10:443 → IP 192.168.1.10 using port 443.

6. Real-World Scenarios & Pitfalls

6.1 Dynamic IP Reassignment

Many ISPs provide dynamic IPs via DHCP which change periodically. If you host a server at home, your address may change making your server unreachable unless you use:

  • Static IP (request from your ISP)
  • Dynamic DNS (DDNS) services which map a hostname to your changing IP automatically

6.2 NAT (Network Address Translation)

Most home networks use private IP ranges (192.168.x.x, 10.x.x.x) and one public IP for the router. NAT translates private addresses to the public IP for outgoing traffic and maps inbound responses correctly.

6.3 Packet Loss & Latency

Network problems include packet loss (dropped packets) and latency (delays). Protocols like TCP help by detecting loss and retransmitting missing packets.

6.4 Security Concerns

Common issues include eavesdropping, man-in-the-middle attacks, and unpatched services. Use TLS/HTTPS, keep servers updated, and follow least-privilege principles for services exposed to the Internet.

7. Basics of Web Development

7.1 Frontend

What users see: structure (HTML), style (CSS), interactivity (JavaScript). HTML defines the page, CSS styles it, and JavaScript manipulates it dynamically.

7.2 Backend

Server-side code handles business logic, authentication, and interacts with databases. Common server tasks: process requests, run application logic, and return responses.

7.3 Databases

Databases store persistent data. Types include relational (SQL) and NoSQL (document stores like MongoDB). Backend servers read/write to these stores.

8. The MERN Stack (Detailed)

  1. MongoDB — A document-oriented NoSQL database storing JSON-like documents. Good for flexible schemas.
  2. Express.js — Lightweight web framework for Node.js that handles routing and middleware.
  3. React.js — A frontend library for building UIs with components and state management.
  4. Node.js — JavaScript runtime that allows running JS on the server.

Example architecture: React app (frontend) → REST/GraphQL API in Express (backend) → MongoDB as data store. Node hosts the server process.

9. What Being a Full-Stack Developer Means

A full-stack developer understands both client-side and server-side development and can:

  • Design and build UI (HTML/CSS/JS/React)
  • Create backend APIs and services (Node/Express)
  • Model and query databases (MongoDB or SQL)
  • Deploy and monitor applications (servers, containerization, CI/CD)

Note: Full-stack does not mean expert in everything — but comfortable across the full pipeline and able to connect the pieces.
